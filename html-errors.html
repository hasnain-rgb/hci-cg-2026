Questions and Answers

Q1: Why is using multiple <h1> tags a mistake?

Answer: Each page should have only one <h1> to define the main topic. Multiple <h1> tags confuse screen readers.

Q2: Why should every <img> have an alt attribute?

Answer: The alt text provides a description for screen readers and when images fail to load.

Q3: What is wrong with using <div> everywhere?

Answer: <div> has no meaning. Semantic tags like <header>, <nav>, and <section> improve accessibility.

Q4: Why is “click here” bad link text?

Answer: Links should describe the destination so they make sense when read alone.

Q5: Why must forms have labels?

Answer: Labels help users understand what to enter and are required for accessibility.

Q6: Why should tables not be used for layout?

Answer: Tables are only for data. Using them for layout harms screen-reader navigation.

Q7: What is wrong with skipping heading levels?

Answer: Jumping from <h1> to <h3> breaks the content hierarchy.

Q8: Why must navigation be inside a list?

Answer: Screen readers expect menus to be structured with <ul> and <li>.

Q9: Why should the <html> tag have a lang attribute?

Answer: It tells screen readers which language to use for pronunciation.

Q10: Why must tags be closed properly?

Answer: Unclosed tags break the document structure and can cause display issues.
