Context: The “Who Paid?” App

The “Who Paid?” app is a group expense application where users want to check who paid last or record a payment. A common usability problem is that users cannot clearly tell who made the most recent payment or what the current system state is.

Applying Norman’s 7 Stages of Action

1. Form Goal

User goal: “I want to check who paid for the last expense.”

The user has a clear goal to find payment information.

2. Form Intention

Intention: “I will open the app and look for the last transaction.”

The user decides to use the app to achieve the goal.

3. Specify Action

Action plan: “I will tap the ‘Recent Payments’ or ‘History’ section.”

The user chooses which interface action to perform.

4. Execute Action

Execution: The user taps buttons or navigates through menus.

If labels or icons are unclear, the user may not know what to tap.

5. Perceive System State

Perception: The app displays a list of transactions.

The user sees names, amounts, and timestamps.

6. Interpret System State

Interpretation: “This entry shows Ali paid the bill yesterday.”

The user tries to understand what the system feedback means.

7. Evaluate Outcome

Evaluation: “Yes, this answers my question about who paid last.”

The user checks whether the goal has been achieved.

Reference: Dix et al., Human–Computer Interaction, Chapter 9

Gulfs of Execution and Evaluation in the “Who Paid?” App

Gulf of Execution

The Gulf of Execution is the gap between what the user wants to do and what actions the interface makes visible or easy to perform.

  • Buttons may have unclear labels (e.g., “Log”, “Update”).
  • Users may not know where to tap to see the last payer.
  • Icons may not clearly represent their function.

Result: The user has the goal but does not know how to perform the correct action.

Gulf of Evaluation

The Gulf of Evaluation is the gap between what the system shows and how easily the user can understand the system’s state.

  • The app shows a list but does not clearly highlight who paid last.
  • No explicit message such as “Last paid by ___”.
  • Timestamps or names may be unclear.

Result: The user performs the action but cannot easily interpret the outcome.

Reference: Dix et al., Human–Computer Interaction, Chapter 9

Design Improvements (Optional)

  • Add a clear label: “Last Paid By: Ahmed”
  • Highlight the most recent transaction
  • Use icons with text labels (e.g., “Recent Payments”)
  • Provide confirmation feedback (“Payment recorded successfully”)
